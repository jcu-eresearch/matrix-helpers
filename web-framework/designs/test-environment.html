Parse statements (Reference)

Design Tags

These Design Tags are a central aspect to making a Design dynamic. Areas can be customised in Design Customisations (or sub-Customisations) and body content can be included, amongst many more things. Consult the reference for full details.

Parse code Output 
<MySource_AREA id_name="metadata" design_area="metadata" />
 
<MySource_AREA id_name="body" design_area="body" />
 

                  <MySource_AREA id_name="settingsId" design_area="declared_vars" print="no">
                    <MySource_DECLARE name="colour" value="#deadbf" type="colour" description="A declared setting"/>
                  </MySource_AREA>
                  <style>
                    .deadbeef {
                      /* The following color definition comes from a PRINT Design Tag */
                      color: <MySource_PRINT id_name="settingsId" var="colour" />;
                    }
                  </style>
                

                    <MySource_AREA id_name="back_to_pages" design_area="access_history">
                      <MySource_SET name="asset_types" value="page_standard" />
                      <MySource_ASSET>
                        <a href="<MySource_PRINT var="asset_link" />">
                          <MySource_PRINT var="asset_short_name" />
                        </a>
                      </MySource_ASSET>
                    </MySource_AREA>
">

Global Keywords (Reference)

Global keyword replacements can be used within various assets in the system to show different information about the current asset the user is viewing, the current site they are viewing or their user account if they are logged in. For example, to show the name of the page the user is currently viewing  as  a  heading  in  your Design, you can use the global keyword replacement %globals_asset_name%.

Site

Keyword Replacement Value Information Shown
%globals_site_<attribute>%

The system does a replacement of "globals_site_" with "asset_" so that "globals_site_assetid" is the same as calling "asset_assetid"

Any attribute of the current Site asset. For example, %globals_site_name% shows the name of the Site the user is currently viewing. This global keyword replacement could then be used in a Design to show the name of the Site. If the name of the Site then changes, the Design will not need to be updated, as the global keyword replacement will source the information directly from the Site.

The following are examples of this in action.
%globals_site_name% %globals_site_name% Name of the current Site asset
%globals_site_url% %globals_site_url% Site URL
%globals_site_assetid% %globals_site_assetid% Site's Asset ID
%globals_site_<keyword>% %globals_site_<keyword>% All other standard asset keywords can be used here. We've demonstrated some above, but any other keywords listed will work as well.
%globals_site_index_id% %globals_site_index_id%

An asset ID of a Site can be appended to these global keyword replacements to specify the Site to return the asset ID values from. For example, the global keyword replacement:

%globals_site_index_id:100%
(value: %globals_site_index_id:100%)

Will return the asset ID of the Index page of the Site with the asset ID of 100.
%globals_site_not_found_id% %globals_site_not_found_id% Site special page ID
%globals_site_archive_id% %globals_site_archive_id% Site special page ID

Asset

Keyword Replacement Information Shown
%globals_asset_attribute_<attribute>% Any attribute of the current front end asset. For example, %globals_asset_attribute_title% (value: %globals_asset_attribute_title%) shows the Title attribute of a Filetype asset the user is currently viewing. This global keyword replacement could then be used in a Design to show the title of the asset. If the title of the asset then changes, the Design will not need to be updated, as the global keyword replacement will source the information directly from the asset.
%globals_asset_attribute_<attribute>:<assetid>% Any attribute of any asset in the system. For example, if we have an Image asset in our system with an asset ID of #67 the global keyword %globals_asset_attribute_title:67% (value: %globals_asset_attribute_title:67%) will show the title of this Image asset, while the global keyword %globals_asset_attribute_size:67% will show its file size.
%globals_asset_metadata_<fieldname>% Any metadata value of the current front end asset. For example, if you have a Description metadata field on your asset, %globals_asset_metadata_Description% will show the value of this field.
%globals_asset_metadata_<fieldname>:<assetid>% Any metadata value of any asset in the system. For example, %globals_asset_metadata_Description:100% will show the Description metadata value on asset #100.
%globals_asset_contents_paint_layout_id_<paint_layout_id>:<assetid>% The contents of any asset in the system with a specific Paint Layout applied where <paint_layout_id> is the asset ID of the Paint Layout and <assetid> is the asset with the contents to print.
%globals_asset_contents_paint_layout_name_<paint_layout_name>:<assetid>% The contents of any asset in the system with a specific Paint Layout applied where <paint_layout_name> is the user defined name of the Paint Layout and <assetid> is the asset with the contents to print.

User-centric keywords

Because users are just another type of Asset, you can utilise the asset-centric keywords to output user details. At the end of the listing, there are also additional global user keywords for groups, messages and so forth.

If an asset is going to be viewed by a user who is logged into your Site, you can personalise the content through the use of global keyword replacements. For example, if the page you are creating can only be viewed by a member of your Site who has logged in, then you can personalise the page with the member's details, such as their name. The format of the global keyword replacement is:

%globals_user_attribute_<fieldname>%
%globals_user_attribute_username% %globals_user_attribute_username% User ID of the current user.
%globals_user_attribute_first_name% %globals_user_attribute_first_name% First name of the current user.
%globals_user_attribute_last_name% %globals_user_attribute_last_name% Surname of the current user.
%globals_user_attribute_email% %globals_user_attribute_email% Email address of the current user.
%globals_user_metadata_<fieldname>% See below for demo

You can also show any metadata values from the user account by using this format, replacing the fieldname with what you'd like displayed. See the next row for a complete view of all metadata available.

For example, if you have applied a Metadata Schema to a user account that contains the field Hobbies, you can show the value that has been entered into this field by using the global keyword replacement %globals_user_metadata_Hobbies%. You can place these keyword replacements within the content of an asset. Squiz Matrix will then replace these keyword replacements with the correct information from the user account.
%globals_user_metadata_user.saml-response% The SAML response for the current logged in user.
%globals_user_metadata_user.student-number% %globals_user_metadata_user.student-number% The student number for the current user.
%globals_user_metadata_user.staff-id% %globals_user_metadata_user.staff-id% The staff ID for the current user.
%globals_user_frontend_metadata% Keyword to output the frontend metadata for this current user asset (undocumented)
%globals_user_member_groups%

The following keyword format can be used to print the user group asset IDs that the current user is a member of
  • %globals_user_messages_inbox%
  • %globals_user_messages_sent%
  • %globals_user_messages_trash%

The keywords can be used to extract internal message data in JSON format from the current user's inbox, sent, and trash message folders

By default, the max limit on messages returned for these keywords is 100. A keyword parameter can be used to change the result limit, for example: %globals_user_messages_inbox_10%. This would return the latest 10 messages from the user's inbox.
%globals_user_sugar_<module_name>_id%

The keyword can be used to print any SugarCRM Record ID applied to the current user

Where <module_name> is the name of the Module that the Record ID is stored against. For example, %globals_user_sugar_Contacts_id%.

Even more globals

Keyword Value Information Shown
%globals_date% %globals_date%

This will display the current server date and time, for example, 6th Feb 2009 5:00pm.

To print a date relative to the current date and time, the following keyword replacement can be used:

  %globals_date_relative_<relative_date>%

where <relative_date> is the relative date value to return, for example, next monday or this friday.

Like the %globals_date% keyword replacement, you can append a PHP date suffix to the end of these keywords to specify the date format to display, for example, %globals_date_relative_this_friday_d-m-Y% yields 

%globals_date_relative_this_friday_d-m-Y%
.
%globals_date_M% %globals_date_Y% %globals_date_M% %globals_date_Y%

This global keyword replacement can be suffixed with a PHP date format (ref: PHP: Date Manual) for greater control of the date and time that is displayed.

%globals_entry_asset% %globals_entry_asset%

The following keyword replacement will display the asset ID that was first accessed in the user's current session.

%globals_session_saml_attributes%

The following global keyword replacements are available to provide access to a SAML user's attributes after being authenticated by a SAML Account Manager.

For more information, refer to the SAML manual.

This will print a JSON string representation of all attributes of the logged-in SAML user.

You can print specific attributes using the following keyword format:

%globals_session_saml_attributes_<attribute>_0%

where <attribute> is the attribute information to print. The index can be set to specify the values to print in the array. In the above example, only the first value in the array will be specified.

Please note that a SAML user's attributes are only stored temporarily for a session. As such, these keywords are only available for the session. Permanent storage can be made available using triggers to access attributes and store them in metadata.

Server Variable

The format of the global keyword replacement for the web server is:

%globals_server_<variable>%

where <variable> is the PHP session variable that you want to return server information from (ref: PHP $_SERVER Manual). For example, the global keyword replacement %globals_server_http_user_agent% will return the header of the current request (e.g. %globals_server_http_user_agent% yields 

%globals_server_http_user_agent%
).

Please note that the server information available will vary depending on the setup of your web server.

Session Variable

The format of the global keyword replacement for a session is:

%globals_session_<variable>%

where <variable> is the name of the item you want to access in the Squiz Matrix session sandbox. This global keyword replacement can be used in conjunction with Triggers. For example, in a Trigger, you can set a session variable and then access it via this global keyword replacement. For more information about Triggers, refer to the Triggers manual.

HTTP GET Variable

The format of the global keyword replacement for a HTTP GET variable is:

%globals_get_<variable>%

where <variable> is the name of the variable that you want to access. Only scalar variables can be used as global keyword replacements. Arrays are not supported.

HTTP POST Variable

The format of the global keyword replacement for a HTTP POST variable is:

%globals_post_<variable>%

where <variable> is the name of the variable that you want to access. Arrays are not supported.
